## Findings

### BLOCKER 1 — Hardcoded postgres password in Git

```yaml
- op: add
  path: /spec/values/global
  value:
    postgresql:
      auth:
        password: risingwave   # ← literal secret in Git
```

`existingSecret: risingwave-pg-credentials` is already set, but the literal `password: risingwave` is committed to the Flux-watched repo. Two problems:
- Audit / CySec / Wiz will flag this as a secret in source control.
- Acts as a silent fallback if `existingSecret` ever fails to resolve — worst kind of compromise mode.

**Fix**: remove the entire `/spec/values/global` add operation. `existingSecret` alone is sufficient.

### BLOCKER 2 (DATA LOSS RISK) — stateStore S3 bucket replacement

```yaml
- op: replace
  path: /spec/stateStore
  value:
    dataDirectory: hummock
    s3:
      bucket: risingwave-state-op-usxpress-dev
      credentials:
        accessKeyRef: AccessKeyID
        secretAccessKeyRef: SecretAccessKey
        useServiceAccount: true
      region: us-east-2
```

Replaces the entire stateStore block. If the running RW currently writes hummock to a different bucket name (e.g., legacy on-prem bucket), this PR points compute pods at `risingwave-state-op-usxpress-dev` — which may not exist OR may be empty.

**Effect**: compute pods come up trying to recover from no hummock → effective data loss for Tim.

**Must verify before approve**:
```bash
# Current bucket on the live RW
kubectl get rw risingwave -n risingwave \
  -o jsonpath='{.spec.stateStore.s3.bucket}{"\n"}'

# Confirm bucket exists + has hummock data in 700736442855 (USX-Dev)
aws s3 ls s3://risingwave-state-op-usxpress-dev/hummock/ --profile usx-dev | head
```

If the live bucket name differs from the patch value, **DO NOT merge** until either:
- Patch updates to match live bucket name, OR
- A migration plan exists (copy hummock data, then switch).

### BLOCKER 3 — risingwave-operator version float

```yaml
- op: replace
  path: /spec/chart/spec/version
  value: ">=0.1.0 <0.2.0"
```

Range selector — Flux/Helm picks the latest 0.1.x on any reconcile. Operator upgrades can trigger CR reconciliation → compute pod restart on Tim.

**Fix**: pin to the exact version currently running:
```bash
kubectl -n risingwave get hr risingwave-operator -o jsonpath='{.spec.chart.spec.version}{"\n"}'
```
Use that exact value (e.g., `0.1.5`).
